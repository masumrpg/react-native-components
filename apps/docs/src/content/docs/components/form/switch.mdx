title: Switch
description: A toggle switch component for binary choices with smooth animations and customizable styling.

The Switch component provides an intuitive way to toggle between two states. It features smooth animations, multiple variants, and follows platform design guidelines.

## Features

- **Smooth Animations**: Fluid transitions with spring physics
- **Multiple Variants**: Support for different color schemes and states
- **Size Options**: Five different sizes from extra small to extra large
- **Customizable Colors**: Custom track and thumb colors
- **Micro-interactions**: Subtle scale animations for better user feedback
- **Accessibility**: Full accessibility support with proper semantics
- **Platform Styling**: iOS-inspired design with shadows and smooth curves

## Basic Usage

```tsx
import { useState } from 'react';
import { Switcher, SwitcherLabel } from '@rnc/theme';
import { View } from 'react-native';

export default function BasicSwitchExample() {
  const [isEnabled, setIsEnabled] = useState(false);

  return (
    <View style={{ flexDirection: 'row', alignItems: 'center' }}>
      <SwitcherLabel position="left">
        Enable notifications
      </SwitcherLabel>
      <Switcher
        value={isEnabled}
        onValueChange={setIsEnabled}
      />
    </View>
  );
}
```

## Variants

The Switch component supports multiple variants for different use cases:

```tsx
import { Switcher } from '@rnc/theme';
import { View } from 'react-native';

export default function SwitchVariantsExample() {
  const [states, setStates] = useState({
    primary: false,
    secondary: false,
    success: false,
    warning: false,
    error: false,
    info: false,
  });

  const updateState = (key: string) => (value: boolean) => {
    setStates(prev => ({ ...prev, [key]: value }));
  };

  return (
    <View style={{ gap: 16 }}>
      <Switcher
        variant="primary"
        value={states.primary}
        onValueChange={updateState('primary')}
      />

      <Switcher
        variant="secondary"
        value={states.secondary}
        onValueChange={updateState('secondary')}
      />

      <Switcher
        variant="success"
        value={states.success}
        onValueChange={updateState('success')}
      />

      <Switcher
        variant="warning"
        value={states.warning}
        onValueChange={updateState('warning')}
      />

      <Switcher
        variant="error"
        value={states.error}
        onValueChange={updateState('error')}
      />

      <Switcher
        variant="info"
        value={states.info}
        onValueChange={updateState('info')}
      />
    </View>
  );
}
```

import { useState } from 'react';
import { Switcher, SwitcherLabel } from '@rnc/theme';

      variant: 'warning' as const,

      key: 'autoBackup' as const,

## API Reference

### Switcher

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `value` | `boolean` | - | Current switch state |
| `onValueChange` | `(value: boolean) => void` | - | Callback when switch state changes |
| `size` | `'xs' \| 'sm' \| 'md' \| 'lg' \| 'xl'` | `'md'` | Switch size |
| `variant` | `'primary' \| 'secondary' \| 'outline' \| 'filled' \| 'ghost' \| 'success' \| 'warning' \| 'error' \| 'info' \| 'destructive'` | `'primary'` | Switch color variant |
| `disabled` | `boolean` | `false` | Whether the switch is disabled |
| `trackColor` | `{ false?: string; true?: string }` | - | Custom track colors for off/on states |
| `thumbColor` | `string` | `'#FFFFFF'` | Custom thumb color |
| `animated` | `boolean` | `true` | Whether to animate state changes |
| `style` | `StyleProp<ViewStyle>` | - | Additional styles |

### SwitcherLabel

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `children` | `React.ReactNode` | - | Label content |
| `position` | `'left' \| 'right'` | `'right'` | Label position relative to switch |
| `style` | `StyleProp<ViewStyle>` | - | Additional styles |

## Best Practices

### Do's

- Use clear, descriptive labels that explain what the switch controls
- Provide immediate feedback when the switch state changes
- Use appropriate variants to convey meaning (success for positive actions, error for destructive actions)
- Consider the context when choosing switch size
- Group related switches together in settings panels

### Don'ts

- Don't use switches for actions that require immediate execution (use buttons instead)
- Don't change switch state without user interaction
- Avoid using too many different variants in the same interface
- Don't make switches too small for touch targets (minimum 44pt)
- Don't use switches for multi-option selections (use radio buttons or pickers)

### Accessibility

- Switches automatically include proper accessibility labels
- Use `SwitcherLabel` to provide descriptive text
- Ensure sufficient color contrast for all states
- Test with screen readers to verify proper announcements
- Consider providing additional context for complex settings

## Troubleshooting

### Common Issues

**Switch not responding to touches**
- Ensure `onValueChange` callback is provided
- Check that the switch is not disabled
- Verify the touch target is large enough

**Animation not working**
- Check that `animated` prop is set to `true` (default)
- Ensure react-native-reanimated is properly installed
- Verify that the component is not being re-rendered excessively

**Custom colors not applying**
- Ensure color values are valid hex, rgb, or named colors
- Check that `trackColor` object has the correct structure
- Verify that custom colors have sufficient contrast

**Performance issues with many switches**
- Consider using `FlatList` for large lists of switches
- Implement proper key props for list items
- Use `React.memo` to prevent unnecessary re-renders
